# LUMEN UNIVERSAL CONNECTION RULE™

Date: 20260610-221225
Status: Hard Rule / Cross-Project
Applies To: Lumen OS, Lumen Website, Vault, ChatGPT/OpenAI Bridge, Future Lumen-Native AI

## Rule
Lumen must be universally connected across:
- Lumen OS
- Lumen website
- Website-only Lumen experience
- Vault
- ChatGPT/OpenAI bridge
- Future Lumen-native AI replacement
- Future Lumen-native OS/runtime/tooling

## Access Channel Rule
Users may access Lumen through different channels:
- Full OS/native experience
- Website-only experience
- Vault-mediated experience
- ChatGPT/OpenAI bridge experience
- Future Lumen-native AI experience

These are not isolated products.
They are connected access channels into the same continuity-governed Lumen ecosystem.

## Website-Only User Rule
Some users may want to use Lumen only through the website.
That must remain a supported path.

Website users still inherit:
- Continuity
- Permissions
- Entitlements
- Vault-backed records
- Accessibility
- Localization
- Security
- Recovery
- User-specific context where consented and appropriate

## OpenAI Bridge Rule
OpenAI/ChatGPT may be used as a bridge layer until Brian can replace it with Lumen-native capability.

OpenAI is not final authority.

The long-term path is:
Bridge → Adapter → Lumen-native replacement

## Vault Rule
The Vault is the continuity memory layer connecting:
- OS
- Website
- User sessions
- Project transfers
- ChatGPT bridge
- Future native Lumen AI

## Preservation Rule
This requirement is hard-coded across projects.
Do not drop it in future transfers.
