## Required Behavior
Every dynamic simulator, policy pack, scorecard, ledger, or gate must preserve:
- No bypassing
- No fake pass
- No hidden exceptions
- Founder/private protection
- User isolation
- Permission safety
- Vault-first evidence
- Auditability
- Recovery path
- Lineage
- Dynamic per-user adaptation
- Continuous validation

## Evidence Rule
Claims of readiness, superiority, or institution-class maturity require:
- Evidence record
- Simulation result
- Validation result
- Scorecard
- Vault reference
- Recovery path
- Founder approval where required

## Living Twin Rule
Living twins must be dynamic, per-user where appropriate, permission-safe, evidence-backed, and continuously improved.

Simulation supports proof.
Simulation does not replace proof.
