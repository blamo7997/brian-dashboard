# REPOSITORY CONSTITUTION™

Date: 20260610-220357
Status: Canonical

## Purpose
Govern the ChatGPT Project Continuity Repository so it can survive many projects without context loss, duplicate authority, orphaned records, or destructive rewrites.

## Constitutional Laws
1. Preservation First
2. Additive First
3. Canonical Authority
4. Delta Over Duplication
5. Lineage Preservation
6. Context Preservation
7. Validation Before Export
8. Transfer Before Close
9. Archive Before Delete
10. Recovery Before Risk
11. Active Layer Before Historical Layer
12. Reference Before Copy
13. No Silent Replacement
14. No Orphan Records
15. No Missing Continuation Points

## Repository Hierarchy
Repository
→ Program
→ Series
→ Container
→ Project
→ Transfer
→ Delta
→ Revision

## Canonical Rule
Only one canonical active record should exist for each subject.

Older generations are preserved as archived, superseded, reference, delta, or revision records.

## Non-Destruction Rule
Important records are never silently deleted.
