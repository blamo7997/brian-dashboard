# Protected Data Authority Model™

Date: 20260611-144150
Project: Lumen Genesis
Status: Active / Hard Rule / Cross-Project

## Core Rule
Anything authorized that cannot or should not live directly in the canonical repository must be routed into the Vault or indexed as a protected Vault reference.

## Unified Authority
Repository + Vault = protected continuity authority.

Repository:
- Active indexed continuity structure
- Architecture indexes
- Validation indexes
- Runtime indexes
- Transfer indexes
- Public-safe continuity artifacts where appropriate

Vault:
- Protected truth source
- Sensitive evidence
- Founder/private data
- User-isolated records
- Authority records
- Protected lineage
- Protected audit trails
- Protected references
- Items too sensitive, large, private, or restricted for direct repository exposure

## Change Control Rule
Repository/Vault data may only be:
- Added
- Modified
- Changed
- Archived
- Deleted

through authorized, permission-safe, evidence-backed workflows.

## Delete Rule
Deletion must prefer:
- Archive before delete
- Immutable lineage
- Audit trail preservation
- Vault evidence preservation

unless Brian explicitly authorizes otherwise.

## Protection Rule
All repository/Vault data must preserve:
- Permissions
- Founder/private boundaries
- User isolation
- Timestamps
- Lineage
- Validation
- Audit trails
- Vault-first evidence

## No-Bypass Rule
No system, user, transfer, runtime, repair, import, export, AI bridge, website, OS, or future native AI process may bypass protected Repository/Vault governance.
