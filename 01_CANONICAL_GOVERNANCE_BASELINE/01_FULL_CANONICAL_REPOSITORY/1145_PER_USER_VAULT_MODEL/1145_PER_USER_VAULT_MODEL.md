# Per User Vault Model™

Date: 20260611-132437
Status: Active / Hard Rule / Cross-Project

## Core Rule
Lumen must not depend on manual new project silos.

Lumen must operate as one continuous automatic stream.

## Applies To
- Repository
- Vault
- Architecture
- Lumen OS
- Website
- Website-version Lumen
- Online Lumen access
- AI / OpenAI bridge
- Future native AI
- Runtime engines
- Validation
- Digital twins
- Living twins
- Security
- Permissions
- Transfers
- Every future user continuity space

## Per-User Rule
Every user should have their own permission-isolated continuity repository/Vault space modeled after Brian's repository.

Each user space must include:
- User-specific repository index
- User-specific Vault records
- User-specific permissions
- User-specific entitlements
- User-specific consent
- User-specific history
- User-specific timestamps
- User-specific lineage
- User-specific validation
- User-specific living twins where appropriate

## Isolation Rule
Users must never access:
- Brian/founder-private records
- Other users' repositories
- Other users' Vault records
- Internal governance authority
- System security controls
unless explicitly authorized.

## Seamless Stream Rule
Lumen should continuously:
- Record
- Timestamp
- Index
- Validate
- Route to Vault
- Update lineage
- Update living twins
- Preserve history
- Continue forward

without needing manual project resets.

## No-Bypass Rule
Continuous automation must never bypass Vault authority, permissions, user isolation, founder boundaries, validation gates, or evidence requirements.
