# CANONICAL AUTHORITY ROOT™

Date: 20260610-225845
Status: Canonical Root / Cross-Project

## Root Principle
Every Lumen action must resolve through:

Identity
→ Authority
→ Permission
→ Entitlement
→ Consent
→ Trust Boundary
→ Data Classification
→ Vault Reference
→ Audit Event
→ Recovery Path

## Root Rules
1. Founder authority is protected.
2. User authority is permission-bounded.
3. Website-only users are first-class but cannot access founder/private systems.
4. OS users cannot bypass website/Vault permissions.
5. Website users cannot access OS founder controls.
6. Vault records are permission-scoped.
7. Lumen may edit website/ecosystem records only under proper authority.
8. Web results must be vetted, verified, and filtered.
9. OpenAI/ChatGPT is bridge/adaptor, not final Lumen authority.
10. Future native replacement must preserve continuity.
11. No scaffold or fake final system is acceptable for live/final systems.
12. Everything remains additive unless modification/deletion is required and justified.
