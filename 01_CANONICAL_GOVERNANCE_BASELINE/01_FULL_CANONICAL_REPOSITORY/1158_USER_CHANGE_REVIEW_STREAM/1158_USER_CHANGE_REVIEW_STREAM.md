# User Change Review Stream™

Date: 20260611-134258
Status: Active / Hard Rule / Cross-Project

## Core Rule
No user should experience lag, latency, reload friction, lost connection, broken continuity, blocked flow, visible error, unresolved red output, skipped warning, or hidden defect.

## Continuous Review Rule
Any change entered by any user must be reviewed by Lumen against:
- Repository
- Vault
- Architecture
- Runtime
- OS
- Website
- Website-version Lumen
- Online Lumen access
- AI bridge
- Future native AI
- Permissions
- Entitlements
- Consent
- Security
- Privacy
- Accessibility
- Localization
- Performance
- Compatibility
- Lineage
- Validation history
- Living twins
- Digital twins

## Automatic Repair Rule
Lumen should automatically:
- Detect problems
- Root-cause problems
- Repair authorized problems
- Re-test repaired work
- Re-validate continuity
- Re-check compatibility
- Re-run proof harnesses
- Re-check red/error output
- Preserve evidence
- Timestamp the change
- Record lineage in Vault

## Permission Rule
Automatic repair must remain permission-safe.
No user change may access:
- Brian/founder-private records
- Other user repositories
- Other user Vault records
- Internal authority
- Restricted security controls
unless explicitly authorized.

## No-Skip Rule
Even tiny defects must not be ignored:
- Red output
- Warning output
- Broken link
- Missing file
- Validation mismatch
- Latency regression
- UX interruption
- Compatibility drift
- Continuity break
- Permission inconsistency
- Vault mismatch
- Lineage gap

## User Experience Rule
The user should experience Lumen as seamless, continuous, stable, fast, and self-healing.
