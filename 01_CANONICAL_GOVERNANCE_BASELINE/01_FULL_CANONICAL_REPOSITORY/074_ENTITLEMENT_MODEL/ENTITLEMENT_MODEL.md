# LUMEN ENTITLEMENT MODEL™

Date: 20260610-222040
Status: Canonical Direction

## Purpose
Preserve user rights, memberships, purchases, services, access, and website/OS availability across all Lumen access channels.

## Entitlement Types
- Website access
- OS access
- Vault access
- Product access
- Service access
- Membership access
- A la carte access
- Accessibility access
- Localization access
- Creator access
- Writer access
- Business access
- Admin/founder access
- Future native AI access

## Required Fields
- EntitlementID
- IdentityID
- Source
- Scope
- ChannelAvailability
- Status
- StartDate
- EndDate
- Evidence
- Validation
- Lineage
- Supersession
- RecoveryPath

## Website-Only Rule
A website-only user can have valid Lumen entitlements without using the OS.

## Preservation Rule
Entitlements must be preserved through deltas and lineage.
Do not silently drop customer rights.
