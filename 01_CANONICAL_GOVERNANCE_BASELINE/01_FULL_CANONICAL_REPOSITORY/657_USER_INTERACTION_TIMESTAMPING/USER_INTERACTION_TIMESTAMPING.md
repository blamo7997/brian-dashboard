# USER INTERACTION TIMESTAMPING™

Date: 20260610-224911
Status: Canonical Direction

## Rule
Lumen should timestamp and store important interactions and actions in the Vault, according to permission, consent, privacy, security, and retention rules.

## Timestamped Events
- User requests
- Lumen responses
- Website interactions
- OS interactions
- Vault writes
- Edits
- Approvals
- Permission decisions
- Web searches
- Vetted result returns
- Recovery events
- Export/import events

## Required Fields
- EventID
- Timestamp
- ActorIdentityID
- Channel
- PermissionContext
- VaultReference
- Action
- Result
- RetentionPolicy
- RecoveryPath
